# Tellama v1.2.1

Tellama v1.2.1 is the commercial-quality friend-testing release. It keeps the v1.2.0 settings and text-size improvements while closing the remaining locally actionable reliability and security findings.

## Highlights

- Hardens native generation, stop handling, cancellation, model preflight, and low-memory recovery.
- Improves encrypted database recovery, resilient model downloads, and user-facing error guidance.
- Strengthens the local API server with verified interface binding, bounded requests, busy responses, timeouts, and safer reconnect behavior.
- Tightens public-web reading against local-network access and unsafe redirects.
- Keeps automatic memory under explicit user control, limits retention, and isolates remembered text as untrusted reference data.
- Separates direct GitHub distribution from the Google Play distribution policy.

## Verification

- 154 direct-distribution unit tests and 154 Play-distribution unit tests passed with no failures or errors.
- Both release variants passed Android Lint and R8/resource shrinking.
- The signed direct APK passed APK Signature Scheme v3 verification.
- An Android 10 physical-device upgrade retained the installed model, 48 conversations, four memories, and the selected serving model.
- Real Gemma 4 generation, client disconnect recovery, screen-off foreground serving, low-memory recovery, LAN authentication, busy handling, and rate limiting were exercised on-device.
- The public v1.2.0 to v1.2.1 in-app update path is the final physical-device release gate before tester handoff.
