# Tellama v1.2.2

Tellama v1.2.2 is a focused commercial-quality reliability release for friend testing. It fixes model replacement decisions and keeps every user-visible serving state aligned with the native runtime.

## Highlights

- Releases the resident model before measuring available RAM and running preflight for a larger target model.
- Distinguishes selected, loading, and serving-ready states in Models, Home, Chat, and the local API server.
- Makes serving-model selection transactional: a failed target load restores both the previous selection and previous runtime.
- Protects native load/unload state transitions from screen or coroutine cancellation races.
- Keeps model-load errors and recovery actions in the Models screen instead of showing contradictory serving states across screens.

## Verification

- 652 clean unit-test executions across Direct/Play Debug/Release completed with no failures or errors.
- Direct and Play release lint, R8/resource shrinking, signed APK generation, and signed Play bundle generation passed.
- The APK passed package/version inspection, 16 KiB zip alignment, and APK Signature Scheme v2 certificate verification.
- On an Android 10 physical device, v1.2.2 installed over v1.2.1 without data loss and preserved 49 conversations, four memories, installed models, and serving selection.
- Real 1.5B → 3B → 4B replacements completed, and the larger targets returned HTTP 200 through the localhost Ollama-compatible API without false memory rejection or a fatal runtime event.

## Updating

Tellama v1.2.1 and later check the public release feed and validate the APK SHA-256, package, higher version, and signing certificate before opening Android's installer. Tellama v1.2.0 users must first install a signed v1.2.1-or-later APK manually because the already-installed v1.2.0 verifier cannot be repaired remotely.
