# Tellama v1.2.4

Tellama v1.2.4 is an on-device inference speed overhaul. It fixes the root causes of slow local generation so replies come back in seconds instead of minutes.

## Highlights

- Ships ARM CPU variant backends (dotprod / fp16 / i8mm / SVE) built with GGML_BACKEND_DL + GGML_CPU_ALL_VARIANTS and selects the fastest at runtime, instead of running a scalar fallback.
- Reuses the KV cache across chat turns by matching the common prefix, so each new turn only decodes new tokens.
- Corrects big-core detection so decode uses the right thread count on modern SoCs.
- Reports decode tok/s separately from prefill, giving an honest generation-speed metric.

## Verification

- Runtime backend selection verified on physical devices (dotprod+fp16 variant chosen) via the `TellamaNative` log tag.
- Clean unit-test executions across Direct/Play × Debug/Release passed with no failures.
- On device, multi-hundred-token replies dropped from minutes to tens of seconds.
