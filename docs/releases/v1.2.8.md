# Tellama v1.2.8

Tellama v1.2.8 is a quality-and-polish release: multi-turn context now survives model switches, two new lightweight models join the catalog, and the chat "thinking" state is an animated Tellama-character badge. It also lands 15 findings from an internal code review.

## Highlights

- **Reliable multi-turn context.** Conversation history is rendered as role-delimited turns per model family (Qwen / Llama / Gemma / Phi), so names and earlier context are retained across turns — including when you switch the serving model mid-chat.
- **Two new models, "New" badge.** Phi-4-mini-instruct Q4_K_M (Microsoft, MIT) and Qwen2.5-Coder-3B-instruct Q4_K_M, localized across 15 languages.
- **Animated thinking indicator.** The static "생각 중…" text is replaced by a bobbing Tellama mascot with staggered dots.
- **Smoother, steadier generation.** Token counting moved off the main thread and the native runtime split into separate model-lifecycle and generation locks, so tokenization no longer contends with an in-flight response. Per-turn token budgeting reduced from O(n²) to O(n).
- **Lighter storage.** Database schema v8 drops an unused column via a tested migration; existing conversations, memories, and installed models are preserved.

## Verification

- Full unit-test suite passing (memory curation, prompt formatting, multi-turn budgeting, catalog integrity).
- Galaxy S10 (Android 12): schema 7→8 migration preserves the full conversation; multi-turn context retained; "New" badge shown; new-model download SHA-256 integrity confirmed.
